# Accessibility audit — 0.1.0

A self-audit of DocketLens against WCAG 2.1 AA. Not a substitute for
external testing; we'll commission one before public launch.

## What's in place

### Global

- **Skip-to-content link** — first focusable element on every route
  (`src/components/skip-to-content.tsx`). Invisible until focused;
  moves keyboard focus to the page's `<main>` element on activation.
- **`<html lang="en">`** with `suppressHydrationWarning` for next-themes.
- **Color tokens** — OKLCH-derived; `--color-fg` against `--color-bg`
  exceeds 4.5:1 in both themes. `--color-fg-muted` exceeds 4.5:1 in
  dark mode and 4.6:1 in light mode (re-checked May 2026).
- **Focus indicators** — `.ring-focus` utility applies a 2px accent
  ring with a token-matched offset; applied to every interactive
  element (buttons, links, inputs, sheet close, sidebar toggle).
- **`prefers-color-scheme`** respected via `next-themes` + the
  ThemeProvider; `disableTransitionOnChange` to avoid the flash of
  unstyled transitions.

### Landmarks

| Route group       | Header                 | Main         | Footer        |
|-------------------|------------------------|--------------|---------------|
| Marketing (`/`, `/pricing`, `/about`, `/blog/*`, `/legal/*`) | `<SiteHeader>` `<header>` | `<main>` per page | `<SiteFooter>` `<footer>` |
| App (`/dashboard`, …)| `<Topbar>` `<header>`   | `<main>` per page | — |
| Auth (`/login`, `/signup`) | Wordmark link          | `<div>` (no main) | small `<p>` copyright |

### Navigation

- **Sidebar** — `<nav aria-label="App">`. Active items get
  `aria-current="page"`. Icon-only collapsed mode uses both
  `aria-label` and `title` so AT and mouse users get the affordance.
- **Site header** — `<nav aria-label="Primary">` with
  `aria-current="page"` on the active link.
- **Mobile drawer** — `<aside role="dialog" aria-modal="true"
  aria-label="App navigation">`, focus is trapped inside while open
  (browser default), Esc closes, route change auto-closes.
- **Command palette** — `<div role="dialog" aria-modal="true">`,
  cmd-k toggles, listbox semantics from `cmdk` library.

### Forms and inputs

- **Search input** (`/search`) — visible-to-AT label via
  `<label className="sr-only" htmlFor="search-q">`; icon is
  `aria-hidden`.
- **Auth forms** — visible labels above each field; placeholder is
  used as a hint, not the only label.
- **Settings forms** — visible labels per field.

### Interactive widgets

- **Tabs** (`src/components/ui/tabs.tsx`) — full ARIA wiring:
  `role="tablist"`, `role="tab"` triggers with `aria-controls` +
  `aria-selected` + roving tabindex, `role="tabpanel"` content with
  `aria-labelledby`.
- **AI exec summary button** — `aria-busy` while loading, generated
  region announced as `role="region" aria-label="AI-generated
  executive brief"`.
- **Court heatmap SVG** — `role="img"` with `aria-label`. Cells are
  not individually focusable today; values are exposed via the
  hover tooltip which mouse users see and AT users can request
  through a planned tabular fallback (queued in 0.2.0).
- **Hover preview popover** (`/search`) — `role="tooltip"`,
  `aria-describedby` on the row; closes on Esc, scroll, resize.

### Motion

- All animations respect short durations (≤300ms) and avoid
  parallax / large translations.
- **`prefers-reduced-motion: reduce`** is honored two ways:
  - `<MotionConfig reducedMotion="user">` at the root layout
    propagates to every framer-motion component
    (`<LiveDocketStream>`, `<CaseResultRow>` popover, etc.).
  - A global CSS `@media (prefers-reduced-motion: reduce)` block
    in `globals.css` collapses every animation and transition to
    ~0ms — covers Tailwind utilities (`animate-ping`, `animate-spin`),
    our custom keyframes, the sidebar width transition, the sheet
    slide, and the heatmap hover tooltip fade.

## Known gaps (queued for 0.2.0)

- **Court heatmap** — tabular fallback for AT users (`<table>` mode
  toggle).
- **Toast** — sonner sets `role="status"`/`role="alert"` itself,
  but we should double-check our success/error variants land in
  the correct live region.
- **Reduced-motion testing** in CI (axe-core run on PR).

## How to verify locally

```bash
pnpm dev
# In another terminal:
npx -y @axe-core/cli http://localhost:3000/dashboard --exit \
  --include 'main' --tags wcag2a,wcag2aa
```

Run this for `/`, `/dashboard`, `/search`, `/dockets/dkt_helios_v_northgate`,
`/watchlists`, `/alerts`, `/settings`, `/pricing`.

## Decision log

- **Native `title=` over a custom Tooltip primitive** for collapsed
  sidebar icons — keeps the JS bundle smaller and is universally
  AT-supported. Trade-off: no styling control, slight delay before
  the OS tooltip appears.
- **Skip link finds `<main>` via JS** rather than enforcing per-page
  `id="main-content"` — eliminates 12+ separate edits and works for
  every route (existing and future) by default.
- **Dark mode is the default** but light mode is fully maintained;
  we never rely on color alone — every state has an icon or text
  signifier alongside.
